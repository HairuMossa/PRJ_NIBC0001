NIBC Firmware Documentation

This document describes the NIBC Firmware software architecture, which follows a layered design to ensure modularity and maintainability.

Layered Architecture Overview

The firmware is divided into specific layers, each with a specific responsibilities and dependencies.
This structure makes it possible to

  • Upper layers (Application) can focus on high-level logic.
  • Lower layers (Interface, Controller) handle direct hardware and system interactions.
  • Dependencies always flow downward (Application -> Controller -> Interface -> System).

Layer Descriptions

1. Application Layer

  • Purpose: Implements the system’s high-level logic and state machines.
  • Responsibilities:
    • Defines operational states (Idle, Running, Error, etc.).
    • Executes the control loop at 10 kHz.
    • Triggers telemetry updates at 10 Hz.

2. Controller Layer

  • Purpose: Encapsulates control algorithms and logic for managing system behavior.
  • Responsibilities:
    • Implements the PID controller.
    • Provides abstraction for telemetry data preparation.
    • Decides when and how to call interface functions.

3. Interface Layer

  • Purpose: Provides hardware-agnostic access to peripherals.
  • Responsibilities:
    • Abstracts drivers for ADC, PWM, CAN, Timers, GPIO.
    • Defines clean APIs for Controller layer to use.
    • Makes Application/Controller code is portable.

4. Lower Layers (System / HAL )

  • Purpose: Contains low-level hardware initialization and STM32Cube HAL bindings.
  • Responsibilities:
    • CubeMX-generated initialization code.
    • Hardware abstraction (clock setup, pin mapping, peripheral init).
    • Provides base functions for Interface layer.

Data & Control Flow

  1. Timer Interrupts (10 kHz / 10 Hz) trigger Application logic.
  2. Application Layer calls into the Controller Layer (e.g., run PID control).
  3. Controller Layer requests data from the Interface Layer (e.g., ADC voltage).
  4. Interface Layer interacts with HAL in the Lower Layer.

Timing Goals

  • Control Loop: 10 kHz (100 µs)
  • PWM Carrier: 20 kHz (50 µs)
  • Telemetry: 10 Hz (100 ms)

These are maintained using synchronized timers (TIM1, TIM2, TIM3).
